# API-Rpi16GB

Backend server for the DHBW Anwendungsprojekt. Runs on a Raspberry Pi 5 (16 GB).

## Stack

- **Zig 0.16.0** — HTTP API server
- **PostgreSQL 16** — sensor data + actuator command storage
- **nginx** — TLS termination, reverse proxy, rate limiting
- **Mosquitto** — MQTT broker, TLS, per-sensor ACL
- **Python controller** — MQTT → DB bridge, actuator dispatch
- **Docker** — container orchestration, two isolated bridge networks (`app-net`, `sensor-net`)

## Repo Layout

```
src/                        Zig source
  main.zig                  entry point, reads secrets, starts server
  server.zig                TCP listener + per-connection worker threads
  router.zig                URL dispatch, JWT validation
  auth.zig                  JWT issue + validate (HS256)
  db.zig                    libpq wrapper (query, exec, parameterised variants)
  handlers/
    health.zig              GET /health
    login.zig               POST /auth/login
    sensor.zig              GET + POST /api/v1/sensor-data
    actuator.zig            POST /api/v1/actuator-command
docker/
  docker-compose.yml
  setup_tls.sh              generates CA + certs for nginx and mosquitto
  set_passwords.sh          sets DB passwords + generates MQTT passwd file
  backend/Dockerfile        multi-stage: downloads Zig, compiles, produces minimal runtime image
  nginx/nginx.conf          TLS termination, rate limiting, subnet whitelist
  postgres/init.sql         schema, DB users, grants
  mosquitto/
    mosquitto.conf          TLS on 8883, no anonymous, passwd + ACL
    acl                     per-sensor and controller topic permissions
    ssl/                    broker.crt, broker.key, ca.crt (generated by setup_tls.sh)
    passwd                  generated by set_passwords.sh, never committed
  controller/
    controller.py           MQTT → postgres bridge; dispatches actuator commands
    Dockerfile
  secrets/                  gitignored — lives only on the Pi
docs/
  architecture.md           system design, network topology, security principles
  setup.md                  install + first-run guide
  api.md                    HTTP API reference
build.zig                   Zig build script (platform-aware libpq include paths)
Makefile                    local dev shortcuts (build, run)
```

## Deploy (on the Pi)

```sh
cd ~/API-Rpi16GB/docker
docker compose up --build -d
```

Docker downloads Zig, compiles the backend, and starts all services. See [docs/setup.md](docs/setup.md) for the full first-time setup.

## Local Dev

```sh
make run
```
